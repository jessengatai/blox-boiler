Setup

Via NPM

You can install via node packet manager with npm i blox-boiler

Sass Include

Download and include the bloxBoiler.scss and bloxBoiler.js files which will allow you to alter the default variables at the top of the file. Remember to call the JS after jQuery.

CSS

Or simply include the bloxBoiler.min.css and bloxBoiler.min.js files. Remember to call the JS after jQuery.

# URL Hash States

If you have an anchor tag that links to #foo and either moves to the matching element or simply updates the hash in the URL, when that hash is active we might want to add a class to our button or some other element on the page. We can achieve this using hash data attributes.

#foo on #bar on

Binds the #foo hash by setting data-hash="#foo".

Adds classes when #foo is active with data-classes-onhash="..."

Adds classes when #foo is inactive with data-classes-offhash="..." 


                

              

              
A noteable downside to managing element states via the URL hash is that you can only have one state for the entire page. However, being able to change the state of elements via the browsers native forwards and back buttons make up for it, especially for simple static HTML sites.

              
Below are all the hash related data attributes

              

                

                  
                    
Description
                    Attribute
                  

                  

                    Bind an element to a hash. 

                      Don't forget the # symbol!
                    		
                    data-hash="#hashname"
                  

                  

                    Classes while hash is active.
                    data-classes-onhash="..."
                  

                  

                    Classes while hash is not active.
                    data-classes-offhash="..."
                  

                  

                    Callback to run when hash is made active. 

                      The current element will be passed as the only parameter.
                    		
                    data-callback-onhash="functionName"
                  

                  

                    Callback to run when hash is made inactive. 

                      The current element will be passed as the only parameter.
                    		
                    data-callback-offhash="functionName"
                  

                

              

              

            


            
            

              
Boxes

              
Rowless columns? Kinda. Blox's "boxes" allows for some pretty awesome out of the box' layouts. Give a parent div a class of 'bx-boxes' and it's direct children classes of 'box' then apply widths via data-attributes - pretty simple.

              
Give the window a resize to see it in action. The default column count is 12, which is changeable with a single line of code in the variables.scss file.

              
NOTE: These data-attributes run from the bottom up. Meaning if you ONLY set `data-width-tny="6"` the box will be 6/12 on all screen sizes from tiny up.
